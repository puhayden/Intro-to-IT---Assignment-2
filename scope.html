In terms of Scopes and Limitations, as a team we believe that we are able to create prototypes of the design of the mobile app and the overall function and interactivity of ‘Downstream’ as a database application. This is due to the time available before submission and the skills that our teams currently have. Later, we discussed Application Programming Interface (API), Independent Developer Environments (IDE) and Cloud computing are the general use of these software systems to implement in ‘Downstream’. These systems would be used as a “set of functionalities provided by a third-party component that is made available to software developers” (Bacchelli, Sawant, para.2) for ‘Downstream’, combining all the tools necessary to run the application and a database to hold the information. However, these cannot be developed currently due to their complexity and the limitation of our team's knowledge about these systems. These programs used would be more long term goals for us to implement in ‘Downstream’ as on average these would take a long time to produce for both mobile apps and desktops applications. These would also need a specialised team that has roles such as web server developers, software developers, cloud strategist, etc, to produce these systems that our team simply does not have the capabilities to do so at present. However, in time to deliver our project to the due date of this assignment as well as our team's current skill levels, our team is able to produce and create prototypes of the general layout of ‘Downstream’ website and mobile app which is functional to use. These mockups will be produced on Figma as it is a prototyping tool which can produce mobile app prototypes. These will include an aesthetic design and layout but also how ‘Downstream’ intends to present its data for users.
